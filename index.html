Completing the connection to GitHub

What We Did Together

  1. Create a folder and add files
  2. Initialize the folder as a local repository
  3. Stage files in the folder and commit changes as needed
  4. Create a GitHub account and login
  5. Identify a GitHub repository as the remote origin for our local repository

What Wasn't Working

The push command and pull commands that would allow us to upload and download content from our two related repositories were not functional because the version histories were different. The Git software could detect that we had different commits on our local machine that did not exist on the remote repository.

The Fix

We needed to give our pull command, with a switch that specifically allows for different histories between our two repositories. This would look like:

git pull --allow-unrelated-histories origin main

If you had existing files in the remote repository that were not identical to the versions in your local repository, Visual Studio Code will then walk you through reviewing the conflicts and accepting one version or the other as the "correct" version. Once complete, you can finalize the connection with one last git commit -m "Final remote synch complete!" command.
